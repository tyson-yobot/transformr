# Phase 03 -- AI Insights Layer

> **Superpower Module Design Specification**
> Status: Ready for Build | Priority: Phase 3

---

## Overview

A screen-aware AI insights engine that surfaces contextual, actionable intelligence on every major screen in the app. Each screen receives up to 3 prioritized insight cards generated by Claude, cached with per-screen TTLs, and presented in a horizontally scrollable card strip. Insights are classified by type (trend, anomaly, coaching, celebration) and include optional deep-link CTAs that navigate the user to the relevant detail screen.

---

## Architecture

- **Single Edge Function** (`ai-insights`) handles all 8 screen types via per-screen prompt dispatch
- **SHA-256 data hashing** of context payloads to avoid regenerating identical insights
- **Stale-while-revalidate caching** -- return cached insight immediately, regenerate in background if >50% of TTL elapsed
- **Per-screen context fetchers** query only the data relevant to each screen
- **Rate limiting** -- max 50 insight generations per user per day
- **Graceful degradation** -- on failure, show last cached insight or hide the section entirely

## Database

### `ai_insights_cache` Table

```sql
CREATE TYPE insight_screen_key AS ENUM (
  'dashboard', 'workout_detail', 'nutrition', 'sleep',
  'weight', 'habits', 'readiness', 'progress'
);

CREATE TYPE insight_type AS ENUM ('trend', 'anomaly', 'coaching', 'celebration');

CREATE TABLE ai_insights_cache (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
  screen_key insight_screen_key NOT NULL,
  insight_type insight_type NOT NULL,
  content JSONB NOT NULL,
  -- content shape: { title: string, body: string, priority: number (1-10), actionCta?: { label: string, route: string, params?: Record<string,string> } }
  data_hash TEXT NOT NULL,
  generated_at TIMESTAMPTZ DEFAULT now(),
  expires_at TIMESTAMPTZ NOT NULL,
  dismissed BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_insights_user_screen ON ai_insights_cache(user_id, screen_key) WHERE NOT dismissed;
CREATE INDEX idx_insights_expires ON ai_insights_cache(expires_at);

ALTER TABLE ai_insights_cache ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users see own insights" ON ai_insights_cache
  FOR ALL USING (auth.uid() = user_id);
```

## Edge Function: `ai-insights/index.ts`

### Handler

```
POST /ai-insights
  Body: { screenKey: ScreenKey, forceRefresh?: boolean }

  1. Authenticate user via Authorization header.
  2. Dispatch to the context fetcher for the given screenKey.
  3. SHA-256 hash the fetched context JSON.
  4. Query ai_insights_cache for matching user_id + screen_key + data_hash WHERE NOT dismissed AND expires_at > now().
  5. If cache hit AND elapsed < 50% of TTL --> return cached insights, fromCache: true.
  6. If cache hit AND elapsed >= 50% of TTL --> return cached insights immediately, trigger background regeneration.
  7. If cache miss OR forceRefresh --> generate new insights synchronously.
  8. Check rate limit: count rows in ai_insights_cache WHERE user_id = ? AND generated_at > now() - interval '1 day'. If >= 50, return cached or empty.
  9. Call Claude with screen-specific system prompt + context JSON.
  10. Parse response, validate JSON structure.
  11. UPSERT into ai_insights_cache.
  12. Return { insights, fromCache: false }.
```

### AI Configuration

- **Model:** `claude-sonnet-4-20250514`
- **max_tokens:** `256`
- **temperature:** `0.7`
- **JSON parse with fallback** -- if response is not valid JSON, discard and return cached/empty

### Error Handling

- 401: Missing or invalid auth token
- 400: Invalid screenKey
- 429: Rate limit exceeded (return cached if available, empty otherwise)
- 500: Unexpected error (return cached if available, empty otherwise)

## System Prompt Templates

Each screen gets a tailored system prompt. All prompts share this structure:

```
You are TRANSFORMR's AI insights engine. Generate a single, actionable insight for the user's {screen_name} screen.

Given: {context JSON}

Rules:
- Be specific with numbers ("Your bench press is up 12% this month" not "You're getting stronger")
- Classify as: trend (pattern over time), anomaly (unusual data point), coaching (actionable advice), celebration (milestone/achievement)
- Priority 1-10 (10 = most important)
- Keep title under 40 characters
- Keep body under 120 characters
- Include actionCta with route if the user should navigate somewhere
- Output valid JSON: { title, body, insight_type, priority, actionCta? }
```

### Screen-Specific Prompt Additions

| Screen | Additional Instructions |
|--------|----------------------|
| `dashboard` | Focus on the single most impactful thing the user should know right now. Prefer celebrations if a streak or PR was hit today. |
| `workout_detail` | Highlight muscle group imbalances, volume trends, or recent PRs. Route CTAs to specific exercises. |
| `nutrition` | Flag macro gaps, suggest next meal focus. Route CTAs to `/(tabs)/nutrition/add-food`. |
| `sleep` | Identify bedtime consistency issues and sleep-workout correlations. Route CTAs to `/(tabs)/goals/sleep`. |
| `weight` | Comment on rate of change sustainability. Celebrate milestones. Route CTAs to `/(tabs)/fitness/progress`. |
| `habits` | Surface habits with declining completion rates. Celebrate streak milestones. Route CTAs to `/(tabs)/goals/habits`. |
| `readiness` | Identify the lowest-scoring component and suggest a targeted recovery action. |
| `progress` | Highlight strength gains and approaching milestones. Celebrate PRs. Route CTAs to `/(tabs)/fitness/progress`. |

## AI Service: `services/ai/insights.ts`

```typescript
import { supabase } from '@services/supabase';
import type { AiInsight, ScreenKey, InsightResponse } from '@app-types/ai';

/**
 * Fetch insights for a given screen. Never throws -- returns empty array on error.
 */
export async function getInsights(screenKey: ScreenKey): Promise<AiInsight[]> {
  try {
    const { data, error } = await supabase.functions.invoke('ai-insights', {
      body: { screenKey, forceRefresh: false },
    });

    if (error) return [];
    return (data as InsightResponse).insights;
  } catch {
    return [];
  }
}

/**
 * Dismiss an insight so it no longer appears.
 */
export async function dismissInsight(insightId: string): Promise<void> {
  await supabase
    .from('ai_insights_cache')
    .update({ dismissed: true })
    .eq('id', insightId);
}

/**
 * Force-regenerate insights for a screen, bypassing cache.
 */
export async function refreshInsights(screenKey: ScreenKey): Promise<AiInsight[]> {
  try {
    const { data, error } = await supabase.functions.invoke('ai-insights', {
      body: { screenKey, forceRefresh: true },
    });

    if (error) return [];
    return (data as InsightResponse).insights;
  } catch {
    return [];
  }
}
```

## Store: `stores/insightsStore.ts`

Zustand with explicit `State`/`Actions` interfaces. MMKV persistence for `dismissedIds` only.

```typescript
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import { mmkvStorage } from './mmkvStorage';
import * as insightsService from '../services/ai/insights';
import type { AiInsight, ScreenKey } from '../types/ai';

interface InsightsState {
  insightsByScreen: Record<ScreenKey, AiInsight[]>;
  loadingScreens: Set<ScreenKey>;
  dismissedIds: Set<string>;
  error: string | null;
}

interface InsightsActions {
  fetchInsights: (screenKey: ScreenKey) => Promise<void>;
  dismissInsight: (id: string) => Promise<void>;
  refreshInsights: (screenKey: ScreenKey) => Promise<void>;
  clearError: () => void;
  reset: () => void;
}

type InsightsStore = InsightsState & InsightsActions;

const initialState: InsightsState = {
  insightsByScreen: {
    dashboard: [],
    workout_detail: [],
    nutrition: [],
    sleep: [],
    weight: [],
    habits: [],
    readiness: [],
    progress: [],
  },
  loadingScreens: new Set(),
  dismissedIds: new Set(),
  error: null,
};

export const useInsightsStore = create<InsightsStore>()(
  persist(
    (set, get) => ({
      ...initialState,

      fetchInsights: async (screenKey) => {
        // Duplicate-request guard
        if (get().loadingScreens.has(screenKey)) return;

        set((state) => ({
          loadingScreens: new Set([...state.loadingScreens, screenKey]),
          error: null,
        }));

        try {
          const insights = await insightsService.getInsights(screenKey);
          set((state) => ({
            insightsByScreen: { ...state.insightsByScreen, [screenKey]: insights },
            loadingScreens: new Set([...state.loadingScreens].filter((k) => k !== screenKey)),
          }));
        } catch (err) {
          set((state) => ({
            loadingScreens: new Set([...state.loadingScreens].filter((k) => k !== screenKey)),
            error: err instanceof Error ? err.message : 'Failed to fetch insights',
          }));
        }
      },

      dismissInsight: async (id) => {
        set((state) => ({
          dismissedIds: new Set([...state.dismissedIds, id]),
        }));
        await insightsService.dismissInsight(id);
      },

      refreshInsights: async (screenKey) => {
        set((state) => ({
          loadingScreens: new Set([...state.loadingScreens, screenKey]),
          error: null,
        }));

        try {
          const insights = await insightsService.refreshInsights(screenKey);
          set((state) => ({
            insightsByScreen: { ...state.insightsByScreen, [screenKey]: insights },
            loadingScreens: new Set([...state.loadingScreens].filter((k) => k !== screenKey)),
          }));
        } catch (err) {
          set((state) => ({
            loadingScreens: new Set([...state.loadingScreens].filter((k) => k !== screenKey)),
            error: err instanceof Error ? err.message : 'Failed to refresh insights',
          }));
        }
      },

      clearError: () => set({ error: null }),
      reset: () => set(initialState),
    }),
    {
      name: 'insights-store',
      storage: createJSONStorage(() => mmkvStorage),
      partialize: (state) => ({ dismissedIds: Array.from(state.dismissedIds) }),
      merge: (persisted, current) => ({
        ...current,
        ...(persisted as Partial<InsightsState>),
        dismissedIds: new Set((persisted as { dismissedIds?: string[] })?.dismissedIds ?? []),
      }),
    }
  )
);
```

## Hook: `hooks/useAIInsights.ts`

```typescript
import { useEffect, useMemo } from 'react';
import { useInsightsStore } from '../stores/insightsStore';
import type { ScreenKey, AiInsight } from '../types/ai';

const MAX_INSIGHTS_PER_SCREEN = 3;

interface UseAIInsightsReturn {
  insights: AiInsight[];
  isLoading: boolean;
  refresh: () => Promise<void>;
}

export function useAIInsights(screenKey: ScreenKey): UseAIInsightsReturn {
  const insightsByScreen = useInsightsStore((s) => s.insightsByScreen);
  const loadingScreens = useInsightsStore((s) => s.loadingScreens);
  const dismissedIds = useInsightsStore((s) => s.dismissedIds);
  const fetchInsights = useInsightsStore((s) => s.fetchInsights);
  const refreshInsights = useInsightsStore((s) => s.refreshInsights);

  // Auto-fetch on mount
  useEffect(() => {
    fetchInsights(screenKey);
  }, [screenKey, fetchInsights]);

  // Filter dismissed, sort by priority desc, cap at 3
  const insights = useMemo(() => {
    const raw = insightsByScreen[screenKey] ?? [];
    return raw
      .filter((i) => !dismissedIds.has(i.id))
      .sort((a, b) => b.content.priority - a.content.priority)
      .slice(0, MAX_INSIGHTS_PER_SCREEN);
  }, [insightsByScreen, screenKey, dismissedIds]);

  const isLoading = loadingScreens.has(screenKey);

  const refresh = async () => {
    await refreshInsights(screenKey);
  };

  return { insights, isLoading, refresh };
}
```

## TTL Strategy

| Screen | TTL | Rationale |
|--------|-----|-----------|
| `dashboard` | 4h | Changes throughout day as workouts/meals are logged |
| `workout_detail` | 24h | Static after a workout is completed and logged |
| `nutrition` | 2h | Changes frequently as meals are added throughout the day |
| `sleep` | 12h | Updates once per night when sleep data is recorded |
| `weight` | 24h | Weight changes at most once per day |
| `habits` | 6h | Updates as habits are checked off during the day |
| `readiness` | 4h | Morning score, may update with wearable data mid-day |
| `progress` | 48h | Slow-changing metrics that aggregate over weeks/months |

## Data Flow

```
Screen mounts with <AIInsightsSection screenKey="nutrition" />
  -> useAIInsights('nutrition') fires useEffect
  -> insightsStore.fetchInsights('nutrition')
  -> Duplicate-request guard check
  -> insightsService.getInsights('nutrition')
  -> supabase.functions.invoke('ai-insights', { body: { screenKey: 'nutrition' } })
    -> Edge function: authenticate user
    -> fetchNutritionContext(userId)
    -> SHA-256 hash context
    -> Check ai_insights_cache for matching hash
    -> If cache hit + fresh: return cached
    -> If cache hit + stale: return cached + background regen
    -> If cache miss: call Claude, parse JSON, UPSERT cache
  -> Return { insights, fromCache }
  -> Store updates insightsByScreen.nutrition
  -> Hook filters dismissed, sorts by priority, caps at 3
  -> AIInsightsSection renders horizontal ScrollView of AIInsightCards

User swipes card right
  -> PanGestureHandler detects threshold
  -> Haptic feedback
  -> FadeOut + SlideOutRight animation
  -> insightsStore.dismissInsight(id)
  -> dismissedIds persisted to MMKV
  -> Card removed from filtered list

User taps CTA button on insight card
  -> router.push(actionCta.route, actionCta.params)
  -> Navigate to target screen
```

## Critical Details

| Concern | Detail |
|---------|--------|
| Cache strategy | SHA-256 hash of context JSON; identical data = identical insight, no wasted tokens |
| Stale-while-revalidate | Return cached immediately if >50% TTL elapsed, regenerate in background |
| Rate limiting | 50 generations/user/day, enforced server-side in Edge Function |
| Token budget | `max_tokens: 256`, `claude-sonnet-4-20250514` |
| Never throws | `getInsights()` and `refreshInsights()` catch all errors, return `[]` |
| Duplicate-request guard | Store tracks `loadingScreens` Set; won't re-fetch if already in progress |
| Persistence | Only `dismissedIds` persisted to MMKV; insight data is always fresh from server |
| Swipe dismiss | Optimistic -- adds to local `dismissedIds` immediately, then updates server |
| Zero-footprint loading | `AIInsightsSection` renders `null` when empty and not loading |
| Security | RLS on `ai_insights_cache`: `auth.uid() = user_id` for all operations |
| Error degradation | AI fails -> show last cached insight; no cached insight -> hide section entirely |
| Offline behavior | Insights are online-only; section hidden when offline; never blocks core workflows |
