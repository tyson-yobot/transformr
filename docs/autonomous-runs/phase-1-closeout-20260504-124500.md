### Issue 1 (P0): Android build failure — RESOLVED

The `expo run:android` build had been broken since the original closeout session. Two independent AAPT2 resource compilation errors were identified and fixed:

1. **Invalid color literal:** `android/app/src/main/res/values/colors.xml` contained `<color name="iconBackground">transparent</color>`. AAPT2 in AGP 8.8.2 does not accept the literal `transparent` — changed to `#00000000`.
2. **Missing splashscreen drawable:** `styles.xml` references `@drawable/splashscreen_logo` but expo prebuild never generated it (no splash `image` in app.json). Created `drawable/splashscreen_logo.xml` as a transparent rectangle placeholder.
3. **Material 1.12.0 pin:** Pinned `com.google.android.material:material` to 1.11.0 via `resolutionStrategy.force` to avoid known AAPT2 incompatibility (expo/expo#34566).

All three files are inside `android/` (gitignored). No non-gitignored files changed for this issue.

## Phase 1 Closeout Final Polish (2026-05-06)

**Symptom:** Two 15-second timeout warnings still fired on emulator
cold launch after the 4d15af3 repair:
- `[notifications] getLastNotificationResponseAsync timed out after 15000ms`
- `[notifications] registerForPushNotifications timed out after 15000ms`

**Root cause:** The original closeout prompt explicitly called for a
`Device.isDevice` early return to skip push registration on
non-physical devices. The closeout-finalize agent applied a different
fix (debug-gating the lifecycle messages and bumping the timeout) but
did not add the Device.isDevice guard. On emulator, where FCM/APNS is
unavailable, push registration always times out after 15 seconds.

**Resolution:** Added `Device.isDevice` guard at both `_layout.tsx`
call sites via a `canReceivePushTokens()` helper:
- `registerForPushNotifications` (Fix 1 useEffect) — entire
  registration block short-circuits with a debug log on non-physical
  devices
- `getLastNotificationResponseAsync` (Fix 3 useEffect) — cold-start
  tap check short-circuits with a debug log on non-physical devices

Real-device behavior is completely unchanged. The `withTimeout`
wrapper and its `notificationsLog.warn` timeout messages remain
intact for physical hardware.
